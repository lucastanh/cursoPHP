Protocolo IP:
	O Protocolo de Internet(IP) é a camada lógica que separa as telecomunicações da internet
	Cada dispositivos na internet recebe um endereço numérico único no mundo para identificá-lo
	As informações são divididas em pacotes e etiquetadas com os endereçõs IP de origem e destino. Os pacotes viajam de forma independente na internet

Sistemas autônomos, BGP, PTTs:
	internet é uma rede de redes

	Numero de Sistema Autonomo - ASN

	provedores de serviço, conteúdo, acesso, trânsito

	A internet é formada por Sistemas autonomos(ASes) redes independentes que se interconectam e colaboram entre si
	o Ponto de Troca de Tráfego(PTTs) servem para interligar muitas redes em um só lugar
	O protocolo Border Gateway Protocol(BGP) serve para que as redes aprendam os endereços ip umas das outras

DNS:
	Domain Name System(DNS)
	a lista de nomes é muito grande então são divididas em domínios de primeiro nível:
		ccTLDs: Country Code Top Level Domains
		gTLDs: Generic Top Level Domains
		servidores autoritativos
	resolver -> recursivo -> raiz -> recursivo -> autoritativo .br -> recursivo -> autoritativo do dominio -> recursivo -> resolver

	O browser envia a requisição para o servidor mas essa requisição não vai diretamente, antes ela passa por uma série de roteadores espalhados pelo mundo, quem se encarrega de descobrir qual o caminho 
em que essa requisição vai percorrer é o protocolo IP.
	O protocolo IP consiste basicamente todos os dispositivos terem um endereço e toda informação que vai ser passada são divididas em pacotes e entiquetados com o endereço de origem 
e o endereço de destino, nesse meio tempo tambem está o DNS.
	O DNS consite em uma lista de endereços IPs e seus nomes, como essa lista é muito grande elas são divididas em country codes ou generic Top Level domains,
o que acontece no DNS é que no nosso computador temos um sistema chamado resolver que consulta um servidor chamado recursivo que normalmente está nos provedores de internet, o servidor recursivo pergunta para o servidor raiz,
o servidor raiz pergunta para o autoritativo do ccTLDs ou gTLDS, estes perguntam para o autoritativo do dominio e então se tem a resposta para o servidor recursivo que manda a resposta para o resolver, nesse processo se obtem
o caminho necessário para o trafego de uma requisição desde a origem que é o nosso navegador até o destino que é algum servidor em outra parte do mundo.
